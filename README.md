# Data Cleansing and Explanatory Data Analysis (EDA)

Data cleansing and Explanatory Data Analysis(EDA) is a part of CRISP DM in the name of data preparation. both need a sufficience business understanding to get useful insight from dataset about which to use and which to delete. here, a little explanation difference between them.
1. Data cleansing or data cleaning is the process of identifying and correcting corrupt, incomplete, duplicated, incorrect, and irrelevant data from a reference set, table, or database.
- for beginner usually dataset only contain missing value and we need to handle it with 2 way case deletion and imputation based on statistik knowledge
2. EDA is applied to investigate the data and summarize the key insights. It will give you the basic understanding of your data, it's distribution, null values and much more. You can either explore data using graphs or through some python functions.
- EDA use more business understanding than data cleaning because we can trow out useless coloumn in dataset if it doesn't help in make an insight for business development
- data visualitazion. we can make a data visualization with 2 correlated variable to build an insiht business development
